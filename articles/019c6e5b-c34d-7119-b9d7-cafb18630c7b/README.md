## ファイル名／URLセーフな日付表現

ISO 8601ないしRFC 3339表現は、特にWindowsでファイル名セーフではない。

- Linux: `/`, `\0`
- macOS (HFS+/APFS): `/`, `\0`, `:`
- Windows (NTFS): `\`, `/`, `:`, `*`, `?`, `"`, `<`, `>`, `|`

また、RFC 3986においてパーセントエンコードなしで使える文字（unreserved characters）は`A-Z a-z 0-9 - . _ ~`のみである。ISO 8601表現に含まれる以下の文字はreservedであり、URLセーフではない。

- `:` 時刻区切り、タイムゾーンオフセット
- `+` タイムゾーンオフセット（例: `+09:00`）
- `,` 小数点（ISO 8601 preferred）
- `/` 反復間隔の区切り（例: `R3/start/end`）

```
$ date --utc --iso-8601=ns
2026-02-18T01:38:54,410693035+00:00
$ date --utc --rfc-3339=ns
2026-02-18 01:38:32.470815316+00:00
```

- 日付、時刻、期間`P1Y2M3DT4H5M6S`
- 週日付`2026-W08-3`
- 反復`R3/...`

使用文字：数字, `-`, `T`, `:`, `,`, `.`, `+`, `/`, `Z`, `W`, `P`, `Y`, `M`, `D`, `H`, `S`, `R`

簡単な選択肢は2つある。

### ISO 8601 basic format

UTCに限れば容易にファイル名セーフかつURLセーフ。[ISO 8601-1 Working Draft (4.2.2.4)](https://www.loc.gov/standards/datetime/iso-tc154-wg5_n0038_iso_wd_8601-1_2016-02-16.pdf)によれば、basic formatでも最小位の時間要素に小数部を付加できる（形式`hhmmss,ss`）。ISO 8601は小数点として`,`を推奨するが`.`も許容しており、`.`はURLセーフである。

```
$ date --utc +%Y%m%dT%H%M%S.%NZ
20260218T014052.746953373Z
```

非UTCの場合、正オフセットでは`+`が残る（例: `20260218T104052.746953373+0900`）。負オフセットの`-`は問題ないが、非対称なのでUTCに限るとしたほうが話が簡単。

### ファイル名／URLセーフでない文字を置換

ここでは`:`を**C**olon、`,`をcomm**A**、`+`をp**L**us、`/`を**F**orward slashに置換して、エンコードとデコードの例を示す。頭文字以外を取っており苦しいけれど、C, A, L, FはISO 8601のどの表現でも使われないため、あらゆる準拠入力に対して安全で可逆と期待できる。

```
$ date --utc --iso-8601=ns | tr ':,+/' 'CALF'
2026-02-18T02C11C24A838203227L00C00
$ echo "2026-02-18T02C11C24A838203227L00C00" | tr 'CALF' ':,+/'
2026-02-18T02:11:24,838203227+00:00
```

UTCに統一してISO 8601 basic formatを用いるのが一番簡単そうだ
